close-required-evidence-contract

決策審查版說明：Issue #39 顯示 validation.requiredEvidence 接受自然語言字串，卻只能由 controller 滿足固定 token，導致 final-verification 進入無法完成的 retry loop。本 change 將 runner 的 runtime contract 收斂成封閉、可機械判定的證據 token。

Why / 問題價值

目前 runner 允許 requiredEvidence 放入任意字串，例如「pnpm test passes」。但 controller validation 只認得 validators-ran、locked-artifacts-unchanged、implementation-diff-present、non-test-diff-present、post-merge-validation-ran、audit-report-present。未知字串永遠無法被滿足，subagent 即使反覆回報成功也只會再次收到 missing evidence follow-up。

這不是 subagent failure，而是 runtime contract mismatch。修正的成功訊號是：新 DAG parse/load 時直接拒絕 unsupported token；舊 persisted invalid state 在 controller validation 變成 blocked 且沒有 followupPrompt。

What Changes / 變更摘要

封閉 token set

新增共享 evidence registry，TypeScript type 不再包含任意 | string。

Schema + parser fail-fast

JSON Schema 使用 enum array；parser 在 DAG 建立與排程前拒絕 unsupported token。

舊狀態防護

validation runner 偵測 persisted invalid token，回傳 blocked，且不產生 subagent follow-up。

Before / After 比較

變更前

  • Schema/type/parser 接受 arbitrary string。
  • Controller 只能滿足固定 token，未知字串永遠 missing。
  • 同一節點可能反覆 self-report complete → validation follow-up → retry。

變更後

  • Runtime DAG load 階段拒絕 unsupported requiredEvidence。
  • 舊 invalid state 會被 blocked，不會要求 subagent 修 runtime contract。
  • 自然語言驗收改放 validators、audit reports、path policy、objective/scope 或 producer trace metadata。

Architecture / Flow

Affected Surfaces / 影響面

  • Core types: GoalValidationEvidenceRequirement 改成封閉 union。
  • DAG parser/schema: unsupported token 在 runtime DAG load 前失敗。
  • Validation runner: persisted invalid state 轉為 blocked diagnostic。
  • Docs/tests/dist: 文件、測試與 package artifacts 同步。

Scope Boundaries / 範圍邊界

本 change 只處理 goal-runner Phase 1：封閉 requiredEvidence runtime contract。它不實作 goal-dag producer 防呆，也不實作 periodic controller audit agent。

禁止事項：不做 transcript matching、不讓 self-report 成為 validation authority、不讀 OpenSpec/PRD/trace sidecar 來滿足 runtime evidence。

Implementation Slices / 實作切片

  1. Evidence registry: 新增 supported token list、set、type guard。
  2. Parser/schema: enum schema + parseRequiredEvidence() fail-fast。
  3. Runtime guard: runControllerValidation() 舊狀態 blocked/no followup。
  4. Docs/tests/dist: 驗證所有 supported/unsupported cases 並更新 package artifacts。

Rollout / Migration

不需要 DB migration。新 DAG 在 parse/load 時拒絕 invalid token；舊 persisted invalid node 在 validation 階段 blocked。有效 DAG 保持 JSON 形狀不變，只是 requiredEvidence 值域變嚴格。

Current Status / 目前狀態

OpenSpec change 已建立為 draft，任務仍未實作。下一步是實作 source changes、跑測試、更新 dist artifacts，然後再做 archive-readiness。

Risk Register / 風險登錄

High-risk filter / 高風險篩選：會造成無限 retry 或錯誤完成判斷的項目標為高風險。

相容性

舊自然語言 requiredEvidence 會被拒絕。緩解：錯誤訊息列出 supported token 與替代欄位。

舊 persisted invalid state

不能只靠 parser。緩解：validation runner 入口 guard blocked/no followup。

跨 repo drift

goal-dag 需另案同步 producer schema。緩解：runner 先 fail-fast，並在 docs 記錄 handoff。

Post-merge evidence

保持既有 integration-owned 語義，測試防止 pre-validation 誤滿足。

Copy Controls / 複製控制

Decision summary: close requiredEvidence to supported controller tokens; reject unsupported runtime DAG values at parse time; block old invalid state without subagent follow-up; do not add transcript matching or periodic audit in this change.
