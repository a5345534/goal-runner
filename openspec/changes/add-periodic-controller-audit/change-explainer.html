add-periodic-controller-audit

Phase 3 follow-up protection layer：新增低頻率 controller audit watchdog，從 structured runtime state 偵測未知 retry-loop / no-progress / cost-spike pattern，並在 critical 時自動 pause goal 避免 token 浪費。

Why

Controller 可以執行 state transitions，但不會週期檢查整體執行健康。Issue #39 的 retry loop 發生時，deterministic loop 只反覆發 follow-up，無法自我診斷為 unhealthy pattern。Phase 1 關閉了已知 invalid contract，但未來未知的 retry-loop / no-progress / cost-spike pattern 仍需要一個低頻率 diagnostic watchdog。

What Changes

Audit path

新增 runGoalControllerAudit()，和 deterministic tick 分離。

Bounded snapshot

從 runtime state 產出 structured snapshot，不含 full transcript。

Safe pause

只有 critical + high-confidence 時自動 pause-goal；其他 actions 需 deterministic confirmation。

Before / After

變更前

Controller tick 只做確定性同步/驗證/整合。No-progress 或 retry loop 可能持續到 token 耗盡或人為發現。

變更後

每 30 分鐘（可配置）audit 檢查 recent window。偵測到 critical retry-loop / cost-spike → 自動 pause goal。Ledger 記錄，monitor 顯示。

Affected Surfaces

  • Core types: GoalControllerAuditOptions, GoalControllerAuditSnapshot, GoalControllerAuditDecision
  • Controller loop: audit scheduling gate after deterministic tick
  • Ledger store: new audit lifecycle event types
  • Monitor UI: audit summary display line
  • Prompts: strict audit model system prompt

Scope Boundaries

本 change 只做 Phase 3 最小可行 audit：snapshot + pause-on-critical。不實作 cap-retries auto-action、mark-node-blocked auto-action、deterministic contract re-check、或 per-goal audit config。

禁止：audit agent 不能 complete / merge / modify code / modify DAG / replan / override deterministic validation。不能用 transcript matching 滿足 evidence。

Rollout / Migration

不需要 DB migration。Audit 預設 disabled 或 interval 30min。新 ledger event types 加到現有 event model。既有 goals 不受影響，只在 deployment 之後 active 的 goal 才會觸發 audit。

Risk Register

High-risk filter：會誤判或浪費 token 的項目標為高風險。

Model produces invalid JSON

緩解：validate & ignore。

Model recommends unsafe action

緩解：only pause-goal auto；all others require confirmation。

Audit cost exceeds tolerance

緩解：bounded snapshot, configurable interval, maxTokensPerAudit。

False positive pause

緩解：needs critical + high-confidence；human can resume。
