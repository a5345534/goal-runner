Why

當前 monitor 第一屏顯示 raw runtime data 的單行 key=value dump，包含冗長的 subagent ID 和 18 行 controller history。Completed goal 會顯示 Health=Blocked。使用者無法快速回答「完成了嗎？健康嗎？問題是什麼？」。

What Changes

Overview model

新增 GoalMonitorOverview 結構化 view model，統一 health/problem/runtime/events 邏輯。

Health 修正

Completed goal 永遠不會顯示 Blocked；新增 Complete / Complete with warnings / Running。

問題摘要

node-centric 短語，不顯示完整 subagent ID。
