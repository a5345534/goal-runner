Why

目前 /goal monitor 第一屏顯示 raw DAG data，不清除分離 runtime 三層狀態：controller session 活動、hidden continuation 生命週期、controller polling 循環。Active session + 故意 suppressed hidden continuation + running poll 的正常狀態容易被誤判為 stuck。

What Changes

Runtime summary

新增 GoalMonitorRuntimeSummary 衍生素型，從既有 adapter state 計算，不引入新持久層。

Pi TUI runtime band

Controller live pane 上方新增 Session / Hidden / Poll / Runners 摘要行。

OpenCode sections

輸出改為 STATUS / RUNTIME / PROGRESS / NEXT ACTION 群組區段。

Before / After

變更前

Monitor 直接顯示 controller events / node list / subagent records。使用者無法從第一屏判斷 goal 是否正常執行中。

變更後

Runtime band 顯示 Session=idle, Hidden=SUPPRESSED(active turn running), Poll=ACTIVE, Runners=2 running。Active session + suppressed continuation 明確不是 failure。

Architecture / Flow

Affected Surfaces

  • Pi monitor: src/adapters/pi/monitor-ui.ts
  • OpenCode monitor: src/adapters/opencode/monitor-ui.ts
  • Runtime data: derived from existing state; no new persistence

Scope Boundaries

只改 monitor UI presentation layer。不改 controller scheduling、hidden continuation launch、subagent lifecycle。不影響 goal-contract / goal-dag / goal-spec。

Rollout

不需要 migration。Display-only change。既有的 row actions（pause/resume/clear/openSession/stop/kill/archive）保持不變。

Current Status

OpenSpec change 已建立為 draft。待實作 Pi/OpenCode monitor 的 runtime band 展示。
